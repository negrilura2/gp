这个问题 **非常典型，而且你这个代码里刚好踩中了 FastAPI + WebSocket + 长循环 的一个“优雅退出”坑**。

你现在的现象是：

> 按 **Ctrl + C** 后，进程不退出（或者卡住很久）。

这不是你电脑问题，而是 **事件循环里还有任务没结束**。

我们一步一步拆。

---

# ✅ 先说结论（核心原因）

你的 `/ws/audio` 里有一个：

```python
while True:
    chunk = await websocket.receive_bytes()
```

这个是：

## ⚠️ 永久阻塞 await

当你 Ctrl+C 时：

* uvicorn 收到 SIGINT ✅
* 尝试 shutdown ✅
* 但事件循环里还有 **未结束的 coroutine**
* shutdown 等待它们结束 ❌
* 结果：进程卡住

---

# ✅ 为什么 HTTP 接口能退出，但 WebSocket 不行？

普通接口：

```python
@app.post("/verify")
```

请求结束 → coroutine 结束 → loop 干净 ✅

---

但 WebSocket：

```python
while True:
```

是一个：

## 无限生命周期任务

只要客户端没断：

```
协程永远活着
```

---

# ✅ Ctrl+C 时真实发生了什么

内部流程：

```
Ctrl+C
 ↓
uvicorn 捕获 SIGINT
 ↓
开始 shutdown
 ↓
关闭 server socket
 ↓
等待所有 task 完成  ← 卡在这里
```

而你的 task：

```
await websocket.receive_bytes()
```

正在等待网络数据。

它不会自动取消。

---

# ✅ 为什么 `WebSocketDisconnect` 没触发？

很多人误解这里。

```python
except WebSocketDisconnect:
```

只在：

```
客户端主动断开
```

才触发。

Ctrl+C ≠ 客户端断开。

---

# ✅ 真正的问题：协程没有响应 Cancel

Python async 退出依赖：

## `CancelledError`

当 shutdown 时：

```python
task.cancel()
```

会向 coroutine 注入：

```python
asyncio.CancelledError
```

但如果代码没正确处理：

👉 coroutine 不会立刻退出。

---

# ✅ 你的代码缺少关键一层

你现在：

```python
except Exception as e:
```

问题来了：

### ❗ CancelledError 在 Python 3.11+ 是 BaseException

它 **不会被 Exception 捕获**。

于是：

```
cancel 信号来了
↓
receive_bytes 仍在 await
↓
loop 等待
↓
进程卡死
```

---

# ✅ 正确写法（工业级修复）

在 websocket handler 最外层加：

```python
import asyncio
```

然后：

```python
@app.websocket("/ws/audio")
async def websocket_audio(websocket: WebSocket):

    await websocket.accept()

    try:
        while True:
            chunk = await websocket.receive_bytes()
            ...
            
    except asyncio.CancelledError:
        logger.info("WebSocket task cancelled (server shutdown)")
        raise   # ⭐ VERY IMPORTANT

    except WebSocketDisconnect:
        logger.info("WebSocket disconnected")

    except Exception as e:
        logger.error(f"WebSocket error: {e}")

    finally:
        try:
            await websocket.close()
        except:
            pass
```

---

## ⭐ 为什么必须 `raise`

很多人漏这一句。

不 raise：

```
cancel 被吞掉
task 仍存活
uvicorn 等待 forever
```

raise 后：

```
task 立即结束
server 正常 shutdown
```

---

# ✅ 第二个隐藏问题（你这个更关键）

你在 websocket 里做了：

```python
stt.transcribe(tmp_path)
svc.verify(tmp_path)
```

这些 **极可能是同步阻塞函数**（CPU/GPU推理）。

如果是：

```
PyTorch inference
Whisper 推理
```

它们会：

## 阻塞 event loop

导致：

* Ctrl+C 延迟
* shutdown 卡住

---

## 工程级正确写法

放进线程池：

```python
import asyncio

loop = asyncio.get_running_loop()

verify_res = await loop.run_in_executor(
    None, svc.verify, tmp_path
)

trans_res = await loop.run_in_executor(
    None, stt.transcribe, tmp_path
)
```

---

### 为什么？

FastAPI websocket 是：

```
单线程 async event loop
```

CPU任务必须丢出去。

这是 **生产级 FastAPI 必修课**。

---

# ✅ 第三个（很多人不知道）

Windows 下 Ctrl+C 对 asyncio 更容易卡。

建议启动 uvicorn 时加：

```python
uvicorn.run(
    "voice_engine.ai_app:app",
    host="0.0.0.0",
    port=9000,
    reload=False,
    loop="asyncio"
)
```

避免自动策略冲突。
