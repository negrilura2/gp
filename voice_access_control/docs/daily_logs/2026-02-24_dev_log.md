## 2. 解决的问题与技术细节 (Technical Details)

### 2.1 问题一：WebSocket 测试脚本无响应 (VAD 误判)
- **现象**：运行 `scripts/test_websocket_agent.py` 向服务端推流时，服务端无任何响应，也不报错。
- **原因分析**：
  - 测试脚本生成的音频是纯正弦波（Sine Wave）。
  - 服务端集成了 VAD (Voice Activity Detection) 模块。
  - VAD 算法（如 WebRTC VAD 或 Silero VAD）通常基于能量和频谱特征，纯正弦波被判定为非语音（Non-Speech）或静音，导致数据被过滤，未进入后续处理流程。
- **解决方法**：
  - 修改测试脚本，生成 **调制白噪声 (Modulated White Noise)**。
  - 使用白噪声作为载波，叠加低频（2-5Hz）的正弦波作为包络（Envelope），模拟人类语音的音节节奏和频谱特性。
  - **结果**：服务端成功识别到语音活动，触发了后续的声纹验证与 STT 流程。

### 2.2 问题二：数据库路径变更导致账号丢失
- **现象**：后端启动后，原有管理员账号无法登录，提示用户不存在。
- **原因分析**：
  - 项目配置 `settings.py` 中将 SQLite 数据库路径从 `backend/db.sqlite3` 变更为了 `data/db.sqlite3`。
  - 新路径下自动生成了空的数据库文件，未包含旧数据。
- **解决方法**：
  - 执行文件复制操作，将原有的数据库文件覆盖到新路径，恢复了用户数据与声纹记录。

### 2.3 问题三：STT 服务 CUDA 库缺失报错
- **现象**：STT 服务启动或转录时报错 `Library cublas64_12.dll is not found`。
- **原因分析**：
  - `faster-whisper` 或 `torch` 默认尝试加载 CUDA 库。
  - 当前环境（Windows）可能未正确配置 CUDA 12.x 运行时，或者只有 CPU 版本。
- **解决方法**：
  - 在 `STTService` 中增加 `_check_cuda()` 方法，通过 `torch.zeros(1).cuda()` 探测可用性。
  - 增加降级机制：一旦捕获到 `cublas` 或 `dll not found` 相关错误，自动切换 `device="cpu"` 并设置 `_cuda_failed` 标志位，防止后续请求无限重试导致服务崩溃。
  - **代码参考**：[stt_service.py](file:///d:/traepg/gp/voice_access_control/voice_engine/stt_service.py)

### 2.4 问题四：FastAPI WebSocket 服务无法优雅退出 (Ctrl+C 卡死)
- **现象**：在终端按 Ctrl+C 停止服务时，进程卡住，无法退出，必须强制杀死。
- **原因分析**：
  - **无限循环**：WebSocket handler 中 `while True` 接收数据，未处理退出信号。
  - **事件循环阻塞**：`svc.verify` (声纹) 和 `stt.transcribe` (语音转录) 是同步 CPU 密集型任务，直接在 `async def` 中调用会阻塞 `asyncio` 事件循环，导致心跳包和退出信号无法及时处理。
  - **CancelledError 被吞**：Python `asyncio` 在 shutdown 时会向任务发送 `CancelledError`，如果代码中用宽泛的 `except Exception` 捕获且不重新 `raise`，任务就不会结束。
  - **Windows 兼容性**：`uvicorn` 在 Windows 上默认的 loop 策略有时与 `asyncio` 冲突。
- **解决方法** (重构 `ai_app.py`)：
  1. **线程池执行**：使用 `loop.run_in_executor(None, func, ...)` 将同步推理任务移出主事件循环。
  2. **显式捕获 CancelledError**：在 `try...except` 块中增加 `except asyncio.CancelledError: raise`，确保任务能响应停止信号。
  3. **指定 Loop 策略**：`uvicorn.run(..., loop="asyncio")`。
  - **代码参考**：[ai_app.py](file:///d:/traepg/gp/voice_access_control/voice_engine/ai_app.py)
