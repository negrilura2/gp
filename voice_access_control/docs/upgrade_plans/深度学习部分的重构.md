# 二、核心思想（非常重要）

## ❗ voice_engine ≠ scripts

这是工业项目最关键的分层。

------

## ✅ `voice_engine/`（库层）

只负责：

- 模型
- 数据集
- loss
- trainer逻辑

这里的代码：

```
永远不能直接 python xxx.py 运行
```

它是 **library**。

------

## ✅ `scripts/`（入口层）

这里只放：

```
程序入口（entrypoint）
```

例如：

```
train.py
evaluate.py
plot_embedding.py
```

运行方式：

```bash
python -m scripts.train
```

------

这就是为什么你现在已经在用 `-m`。

你其实已经走在正确路上了。

------

# 三、真正的大厂训练入口长什么样

## scripts/train.py

结构几乎永远是：

```python
def main(cfg):

    # 1️⃣ 数据
    train_loader, val_loader = build_dataloader(cfg)

    # 2️⃣ 模型
    model = build_model(cfg)

    # 3️⃣ trainer（训练控制器）
    trainer = Trainer(model, cfg)

    # 4️⃣ 开始训练
    trainer.fit(train_loader, val_loader)


if __name__ == "__main__":
    cfg = load_config()
    main(cfg)
```

注意：

### ⭐ train.py 不写训练细节

训练细节在：

```
voice_engine/trainer.py
```

# 四、真正的训练命令（工业真实）

工程师不会改代码。

他们改配置：

```bash
python -m scripts.train --config configs/ecapa.yaml
```

------

## ecapa.yaml 示例

```yaml
model:
  emb_dim: 192

dataset:
  feature_type: mfcc_delta

training:
  batch_size: 64
  epochs: 80
  lr: 0.001
```
