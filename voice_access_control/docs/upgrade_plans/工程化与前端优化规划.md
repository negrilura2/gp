# 项目升级规划：工程化与前端优化

根据最新的项目重构计划，本项目将遵循 **"深度学习优先 -> 工程化跟进"** 的路线。在完成深度学习核心实验（特征对比、噪声鲁棒性、可视化、归一化）后，将全面转向系统工程化与前端体验提升。

## 📅 阶段一：深度学习核心（进行中）
*参考 `docs/deep_learning_upgrade_plan.md`*
1.  **Level 1-2**: 基线模型与特征对比（已完成）
2.  **Level 3**: 噪声鲁棒性实验（已完成）
3.  **Level 5**: 嵌入空间可视化（**Next Step**）
4.  **Level 6**: 分数归一化（**Next Step**）

## 📅 阶段二：工程化升级 (Infrastructure)
*目标：打造“企业级”IoT + AI 系统架构*

### 1. Docker 全面容器化
- [ ] **Backend**: 编写 Dockerfile，容器化 Django 应用。
- [ ] **AI Service**: 将推理服务独立为微服务容器（如使用 FastAPI 或 TorchServe）。
- [ ] **Database**: 使用 Docker Compose 编排 PostgreSQL/MySQL。
- [ ] **Orchestration**: 编写 `docker-compose.yml` 实现一键启动 (`git clone` -> `docker compose up`)。

### 2. 消息队列解耦 (Kafka/Redpanda)
- [ ] 引入消息队列处理音频推理请求，解耦 API 与 AI 计算。
- [ ] 流程：`API` -> `Producer` -> `Queue` -> `Consumer (AI Service)` -> `DB`。
- [ ] 解决高并发下的推理延迟与阻塞问题。

### 3. Kubernetes 部署 (Optional/Advanced)
- [ ] 编写 Deployment, Service, ConfigMap 配置。
- [ ] 实现 AI 服务的独立扩缩容 (HPA)。
- [ ] 演示滚动更新 (Rolling Update) 与自愈能力。

### 4. CI/CD 流水线
- [ ] 搭建 GitHub Actions。
- [ ] 实现：代码提交 -> 自动测试 -> 构建镜像 -> 推送仓库。

## 📅 阶段三：前端体验优化 (Frontend Polish)
*目标：借助 Skills 提升 UI 科技感与交互体验*

### 1. 科技感 UI 升级
- [ ] **动态波形图**: 使用 Canvas/WebAudio API 在录音时展示实时动态波形（类似 Siri 效果）。
- [ ] **数据可视化**: 使用 ECharts 升级 Dashboard，展示炫酷的实时验证统计、声纹分布图。
- [ ] **交互动效**: 增加验证通过/失败的酷炫动画与音效。

### 2. 模拟器开发 (Device Simulator)
- [ ] 开发一个简单的“门禁设备模拟器”（Web 或 CLI），模拟 IoT 设备发送心跳与音频流。
- [ ] 演示完整的“端-边-云”协同流程。

## 📌 后续步骤细化（结合终极版路线）

### 总体执行顺序
1.  AI 服务拆分
2.  Docker 化与一键部署

4.  消息队列解耦
5.  Celery 异步任务
6.  Gateway 与限流
7.  CI/CD 自动化
8.  K8s（可选）

### Phase 1：深度学习核心强化（承接 Level 5/6）
- **目标**：可解释、可展示、可迁移
- **交付物**：
  - [ ] 嵌入空间可视化脚本（t-SNE/UMAP）与结果图
  - [ ] 分数归一化方案（Z-Norm/S-Norm）与对比评估
  - [ ] 评估图表规范化输出（统一目录、命名、可复现命令）

### Phase 2：AI 服务拆分（最关键）
- **目标**：Django 与推理解耦，形成独立 AI Service
- **步骤**：
  - [ ] 抽取统一推理入口（加载模型、特征提取、打分、阈值判断）
  - [ ] 定义 HTTP 接口（/health、/embed、/verify）
  - [ ] 增加模型版本与特征元数据返回
  - [ ] 在 Django 侧改为 HTTP 调用

### Phase 3：Docker 化与一键启动
- **目标**：`git clone -> docker compose up` 可运行
- **步骤**：
  - [ ] Backend Dockerfile
  - [ ] AI Service Dockerfile
  - [ ] 数据库容器化（MySQL/PostgreSQL）
  - [ ] docker-compose.yml 编排与健康检查

### Phase 4：Redis 与鉴权统一
- **目标**：缓存与认证工程化
- **步骤**：
  - [ ] Redis 引入（登录状态、门禁状态、限流计数）
  - [ ] JWT + Redis blacklist 统一鉴权
  - [ ] 后端缓存策略与过期机制

### Phase 5：消息队列解耦
- **目标**：削峰填谷、提升并发稳定性
- **步骤**：
  - [ ] 接入 Kafka/Redpanda
  - [ ] Producer/Consumer 解耦推理请求
  - [ ] 失败重试与消息堆积监控

### Phase 6：异步任务与系统治理
- **目标**：具备企业级治理能力
- **步骤**：
  - [ ] Celery 处理日志写入、延迟任务
  - [ ] Gateway 限流与降级策略
  - [ ] 统一日志与指标采集

### Phase 7：CI/CD 与云原生加分项
- **目标**：工程闭环与可展示能力
- **步骤**：
  - [ ] GitHub Actions 流水线（test -> build -> deploy）
  - [ ] K8s 基础部署示例（可选）

## 🛠️ 技术栈选型建议
| 模块 | 技术 |
| :--- | :--- |
| **Backend** | Django REST Framework (现有) |
| **AI Service** | PyTorch, FastAPI (建议独立) |

| **Container** | Docker, Docker Compose |
