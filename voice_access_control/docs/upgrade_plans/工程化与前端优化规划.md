# 项目升级规划：工程化与前端优化

根据最新的项目重构计划，本项目将遵循 **"深度学习优先 -> 工程化跟进"** 的路线。在完成深度学习核心实验（特征对比、噪声鲁棒性、可视化、归一化）后，将全面转向系统工程化与前端体验提升。

## 📅 阶段一：深度学习核心（进行中）
*参考 `docs/deep_learning_upgrade_plan.md`*
1.  **Level 1-2**: 基线模型与特征对比（已完成）
2.  **Level 3**: 噪声鲁棒性实验（已完成）
3.  **Level 5**: 嵌入空间可视化（**Next Step**）
4.  **Level 6**: 分数归一化（**Next Step**）

## 📅 阶段二：工程化升级 (Infrastructure)
*目标：打造“企业级”IoT + AI 系统架构*

### 1. Docker 全面容器化
- [ ] **Backend**: 编写 Dockerfile，容器化 Django 应用。
- [ ] **AI Service**: 将推理服务独立为微服务容器（如使用 FastAPI 或 TorchServe）。
- [ ] **Database**: 使用 Docker Compose 编排 PostgreSQL/MySQL。
- [ ] **Orchestration**: 编写 `docker-compose.yml` 实现一键启动 (`git clone` -> `docker compose up`)。

### 2. 消息队列解耦 (Kafka/Redpanda)
- [ ] 引入消息队列处理音频推理请求，解耦 API 与 AI 计算。
- [ ] 流程：`API` -> `Producer` -> `Queue` -> `Consumer (AI Service)` -> `DB`。
- [ ] 解决高并发下的推理延迟与阻塞问题。

### 3. Kubernetes 部署 (Optional/Advanced)
- [ ] 编写 Deployment, Service, ConfigMap 配置。
- [ ] 实现 AI 服务的独立扩缩容 (HPA)。
- [ ] 演示滚动更新 (Rolling Update) 与自愈能力。

### 4. CI/CD 流水线
- [ ] 搭建 GitHub Actions。
- [ ] 实现：代码提交 -> 自动测试 -> 构建镜像 -> 推送仓库。

## 📅 阶段三：前端体验优化 (Frontend Polish)
*目标：借助 Skills 提升 UI 科技感与交互体验*

### 1. 科技感 UI 升级
- [ ] **动态波形图**: 使用 Canvas/WebAudio API 在录音时展示实时动态波形（类似 Siri 效果）。
- [ ] **数据可视化**: 使用 ECharts 升级 Dashboard，展示炫酷的实时验证统计、声纹分布图。
- [ ] **交互动效**: 增加验证通过/失败的酷炫动画与音效。

### 2. 模拟器开发 (Device Simulator)
- [ ] 开发一个简单的“门禁设备模拟器”（Web 或 CLI），模拟 IoT 设备发送心跳与音频流。
- [ ] 演示完整的“端-边-云”协同流程。
