# 二、为什么要引入这些技术（面试关键逻辑）

你以后必须这样解释，而不是“我学了”。

------

## ✅ Kubernetes —— 解决什么问题？

问题：

> AI 推理服务资源消耗大，需要独立扩缩容。

解决：

```
模型服务 = 独立 Pod
自动重启
水平扩展
```

你要表达的是：

👉 **调度问题，而不是部署问题。**

------

## ✅ Kafka —— 为什么不是直接 HTTP？

真实问题：

声纹识别是：

```
请求 → 重计算 → 延迟不稳定
```

如果同步调用：

- API 被阻塞
- 高并发崩溃

Kafka 作用：

```
解耦请求 与 AI计算
```

流程变成：

```
用户请求
→ 写入 Kafka
→ AI服务异步消费
→ 返回结果
```

这是企业级设计思路。

------

## ✅ CI/CD —— 为什么需要？

问题：

AI模型会更新。

如果手动部署：

- 环境错乱
- 服务中断

CI/CD：

```
git push
→ 自动测试
→ 构建镜像
→ 自动部署K8s
```

证明：

👉 你理解软件生命周期。

## 

### Docker 全面容器化

你必须先做到：

```
backend
model-service
db
```

全部 docker 化。

否则后面全部失败。

## 

### 引入 Kafka（最小可行）

不要一开始搞集群。

只做：

```
docker-compose kafka
```

实现：

1️⃣ backend 发布消息

```
produce("voice.verify", audio_id)
```

2️⃣ AI service 消费：

```
consume → 推理 → 写DB
```

面试价值极高。

------

## 

### Kubernetes 部署（核心）

只学必要部分：

你只需要会：

- Deployment
- Service
- ConfigMap
- HPA（可选）

部署：

```
backend deployment
ai-service deployment
kafka deployment
```

推荐：

👉 minikube 或 k3d（本地即可）

## 

### CI/CD Pipeline

推荐最简单路线：

GitHub Actions。

实现：

```
push main
↓
run test
↓
build docker image
↓
push dockerhub
↓
kubectl apply
```

你已经进入真实工程流程。
